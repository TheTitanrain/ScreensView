## CRITICAL

| # | Domain | File:Line | Issue | Fix |
|---|--------|-----------|-------|-----|
| C1 | Security | `ViewerUpdateService.cs:190-201` | Downloaded update `.exe` executed without signature/hash verification → supply chain attack | Verify SHA-256 against pinned checksum or code-sign releases |
| C2 | Reliability | `ScreenshotService.cs:164` (both agents) | `pipeServer.WaitForConnectionAsync().Wait()` — sync-over-async → deadlock risk | Replace with `await` |
| C3 | Reliability | `HttpsBindingManager.cs:46-47` | `.Result` on tasks inside sync method, no timeout → UI thread block | Use `Task.WaitAll()` with timeout |
| C4 | Windows | `ScreenshotService.cs:99-101` (both agents) | `DuplicateTokenEx` called with `SecurityImpersonation` instead of `SecurityDelegation` for primary token creation → may produce invalid token under strict UAC | Change to `SecurityDelegation` (value = 3) |
| C5 | Windows | `ScreenshotService.cs:168,174` (both agents) | `WaitForSingleObject` return value ignored → handles closed while process still running | Check return, only close handles if `WAIT_OBJECT_0` |

## Tests

**Currently covered well:** storage encryption, DPAPI, `RemoteAgentInstaller` OS detection (21 tests), `ApiKeyMiddleware` auth paths (~200 tests total).

**Critical gaps:**

| Gap | Risk | Blocker |
|-----|------|---------|
| Cert pinning callbacks in `AgentHttpClient` | Critical — security logic untested | `HttpClientHandler` not injectable |
| Polling loop (`ScreenshotPollerService`) | High — interval, cancellation, error recovery | `Dispatcher.Invoke` hard dependency on WPF runtime |
| `SessionLockedException` extraction from 503 body | Medium | Need injectable `HttpMessageHandler` |
| Concurrent `ComputerStorageService` writes | Medium | None — write test today |

**To unlock most tests:** extract `IUiDispatcher` (unblocks ~10 polling tests) + inject `HttpMessageHandler` (unblocks cert pinning tests).
