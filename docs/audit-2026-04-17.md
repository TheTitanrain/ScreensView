## LOW / INFO

| # | Domain | Issue | Status | Implementation note |
|---|--------|-------|--------|---------------------|
| L1 | Security | No rate limiting on `/screenshot` endpoint — DoS possible | Resolved | Both agents now use a shared single-flight capture gate. Overlapping capture attempts fail fast with HTTP `429 Too Many Requests` instead of spawning another expensive screenshot pipeline. |

| L4 | Architecture | Magic number `120` seconds duplicated in `LlmCheckService.cs:156` and exception message at 215 — extract to `const` | Resolved | `LlmCheckService` now uses named timeout constants and a shared timeout message template instead of duplicated literals. |
| L5 | Architecture | `MainViewModel.cs:16-19` — static `LocalizationService.Get()` calls in static getters don't refresh on language switch | Resolved | Localized status/error text now stores resource keys and resolves them on access, so language changes refresh the UI text instead of leaving cached strings behind. |
| L6 | Reliability | `LlamaServerProcessService.cs:191` — `TcpListener` not explicitly disposed in `FindFreePort()` | Resolved | `FindFreePort()` now wraps the `TcpListener` in `using var`, and a regression test verifies the returned port can be rebound immediately. |

| L9 | Windows | `Program.cs` (both) — JPEG quality arg parsed but not clamped `0-100` — use `Math.Clamp` | Resolved | Shared helper `ScreenshotQuality.ParseOrDefault()` clamps JPEG quality to `0..100` for both the modern and legacy agent helpers. |
