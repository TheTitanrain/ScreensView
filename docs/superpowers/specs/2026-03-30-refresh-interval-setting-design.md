# Сохранение интервала обновления Viewer — дизайн-документ

**Дата:** 2026-03-30
**Проект:** ScreensView.Viewer (WPF, net8.0-windows)

---

## Контекст

Сейчас слайдер интервала обновления управляет polling только в памяти текущего запуска. После перезапуска Viewer значение возвращается к дефолтным `5` секундам, даже если пользователь выбрал другой интервал.

---

## Дизайн

### Хранение настройки

Используется существующий файл пользовательских настроек:

- путь: `%AppData%\ScreensView\viewer-settings.json`
- модель: `ViewerSettings`
- новое поле: `RefreshIntervalSeconds`

Интервал хранится в секундах как целое число.

### Поведение `MainViewModel`

- При создании `MainViewModel`:
  - загрузить `ViewerSettings`
  - взять `RefreshIntervalSeconds`
  - если значение отсутствует или выходит за диапазон `1..60`, использовать дефолт `5`
  - присвоить результат свойству `RefreshInterval`
- При изменении `RefreshInterval`:
  - сохранить новое значение в `ViewerSettingsService`
  - если polling уже запущен, перезапустить `ScreenshotPollerService` с новым интервалом

### Диапазон значений

- UI уже ограничивает слайдер диапазоном `1..60`
- модель и загрузка настроек должны дополнительно нормализовать значение, чтобы Viewer устойчиво переживал ручное редактирование JSON или старые файлы настроек

### Обратная совместимость

- Старые `viewer-settings.json`, где нет `RefreshIntervalSeconds`, остаются валидными
- В таком случае Viewer использует дефолт `5`

## Проверка

1. При старте Viewer интервал берётся из `viewer-settings.json`.
2. При отсутствии настройки используется `5`.
3. При изменении слайдера новое значение сохраняется в `viewer-settings.json`.
4. Если polling активен, он продолжает работу уже с новым интервалом.
5. `dotnet test` проходит без ошибок.
