# Индикация отключенного компьютера в карточке Viewer — дизайн-документ

**Дата:** 2026-04-01
**Проект:** ScreensView.Viewer (WPF, net8.0-windows)

---

## Контекст

Сейчас признак `IsEnabled` используется только для того, чтобы poller не опрашивал отключённые компьютеры.
Сама карточка в `MainWindow` и footer в `ScreenshotZoomWindow` опираются только на `ComputerViewModel.Status`.

Из-за этого компьютер, отключённый в окне «Управление компьютерами», не получает явной индикации в карточке:

- он может оставаться в `Unknown`
- он может визуально выглядеть как обычная карточка с последним скриншотом
- пользователь не видит разницу между «компьютер отключён вручную» и «компьютер недоступен / ещё не опрошен»

---

## Решение

Вводится отдельное presentation-состояние `Disabled` в `ComputerStatus`.

Это состояние означает только одно: компьютер присутствует в списке, но отключён пользователем через `IsEnabled = false`, поэтому Viewer не должен его опрашивать.

### Поведение

- при создании `ComputerViewModel` из `ComputerConfig` со значением `IsEnabled = false` карточка сразу получает статус `Disabled`
- при изменении компьютера через `MainViewModel.UpdateComputer(...)` переключение `IsEnabled` должно немедленно переводить карточку:
  - в `Disabled`, если компьютер отключён
  - в `Unknown`, если компьютер был снова включён и ещё не получил новый скриншот
- `UpdateScreenshot(...)`, `SetError(...)` и `SetLocked(...)` не должны переопределять `Disabled`, если компьютер выключен вручную
- если ответ от poller приходит уже после ручного отключения компьютера, такой поздний ответ игнорируется

### UI-подача

- в `MainWindow.xaml` карточка получает отдельный цвет индикатора для `Disabled`
- в `ScreenshotZoomWindow.xaml` footer получает ту же индикацию, чтобы состояния не расходились между окнами
- tooltip имени компьютера должен показывать понятное сообщение, почему карточка не обновляется

### Что не меняется

- poller по-прежнему не опрашивает `IsEnabled == false`
- последний успешный скриншот не очищается автоматически
- логика открытия zoom-окна не меняется
- новые сервисы и отдельное хранилище статусов не добавляются

## Проверка

1. Карточка компьютера с `IsEnabled = false` сразу отображается как отключённая.
2. После выключения компьютера через «Управление компьютерами» карточка меняет индикацию без перезапуска Viewer.
3. После повторного включения карточка выходит из `Disabled` и ждёт нового poll как `Unknown`.
4. `dotnet test ScreensView.slnx` проходит без регрессий.
