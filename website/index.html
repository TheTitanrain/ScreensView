Локальная сеть

Скриншоты и управление остаются внутри вашей инфраструктуры.

HTTPS + API-ключ

Агент отдаёт данные по TLS и принимает только авторизованные запросы.

Windows first

Viewer на .NET 8, modern agent и legacy path для Windows 7 SP1.

Self-hosted

Никакой внешней панели и никаких обязательных облачных зависимостей.

Core capability

Стена живого мониторинга, а не просто набор отдельных подключений.

Viewer опрашивает все включённые машины и показывает скриншоты в адаптивной сетке. Карточки сразу отображают состояние: обычный экран, блокировка, LLM-статус, ошибки загрузки или отсутствие активной пользовательской сессии.

Один обзорный экран

Держите десятки рабочих станций в одной панели и открывайте любой экран в zoom-окне по двойному клику.

  • Адаптивная сетка для больших и средних стен мониторинга
  • Контекстные действия прямо с карточки компьютера
  • Состояние locked/disabled видно без отдельной диагностики

Управление без переключения между утилитами

Список компьютеров, обновление, запуск внеочередных проверок и изменение настроек живут внутри одного Viewer.

  • Массовые операции по multi-select или по всему активному набору
  • Обновить сейчас и LLM сейчас без изменения фоновых интервалов
  • Переключение источника подключений и защищённый внешний файл
Operations

Развёртывание и обслуживание агентов встроены в рабочий процесс оператора.

ScreensView не ограничивается просмотром. Viewer помогает добавить компьютеры пачкой, выбрать payload по версии Windows и обслуживать агентские службы без ручной сборки на каждой машине.

1

Добавьте новые машины

Одиночный диалог, список хостов или диапазон IP. API-ключи генерируются автоматически.

  • Режимы «По хостам» и «По диапазону IP»
  • Описание экрана можно задать сразу в карточке компьютера
2

Установите агент удалённо

Viewer копирует payload, создаёт службу и подбирает modern или legacy агент под целевую ОС.

  • Поддержка Windows 10/11 и compatibility path для Windows 7 SP1
  • Пошаговый лог установки, удаления и обновления
3

Держите парк актуальным

Обновляйте агентские службы массово и быстро проверяйте, что экраны соответствуют ожиданиям.

  • Обновить агентов для всех включённых компьютеров
  • Viewer сам проверяет новые релизы приложения
Optional intelligence

LLM-проверка - дополнительный слой контроля, а не обязательный режим.

Если нужно сверять не просто наличие картинки, а ожидаемую визуальную структуру экрана, Viewer может локально запускать multimodal-проверку и маркировать карточки как совпадающие или подозрительные.

Проверка ожидаемого состояния

Описываете стабильную структуру экрана, а Viewer запускает локальную проверку по таймеру или вручную.

  • Статусы `?`, `LLM`, `...`, `✓`, `✗`, `!` видны прямо на карточке
  • Кэш повторяющихся кадров снижает лишние вызовы модели
  • Ручной запуск доступен для всей стены или для одной машины

Локальные модели и диагностируемый пайплайн

Модель и backend живут рядом с Viewer, а все ошибки и сырые ответы остаются в локальном логе.

  • Скачивание совместимой GGUF-модели из настроек
  • Отдельная диагностика для `llama.cpp`, projector и unsupported architecture
  • Fast-profile для скриншотов и ограничение image token budget
Reliability and security

Сайт демонстрирует возможности, но продукт проектировался под эксплуатацию.

ScreensView учитывает реальное поведение Windows Service, secure desktop и особенности удалённого администрирования, а не только «идеальный» сценарий живого скриншота.

Понятные штатные состояния

  • Заблокированная рабочая станция возвращает HTTP 503 и отображается как `Locked`, а не как сломанная машина
  • Выключенные компьютеры исключаются из опроса и массовых операций
  • Viewer пишет причину проверки, пропуска и ошибок в `viewer.log`

Минимально необходимый perimeter

  • TLS 1.2+, self-signed сертификат и pinning thumbprint в Viewer
  • Авторизация по `X-Api-Key`
  • Удалённая установка требует явных прав локального администратора и доступа к `Admin$` / WMI

FAQ

Коротко о главном перед запуском.

Это не полная документация, а обзор типовых вопросов перед первым запуском.

Нужно ли отдельное облако или брокер?

Нет. Viewer и агент общаются напрямую по HTTPS внутри вашей сети.

Поддерживается ли Windows 7?

Есть legacy path для Windows 7 SP1 с `.NET Framework 4.8`, но это compatibility-сценарий, а не современная поддерживаемая платформа.

Что происходит при блокировке рабочей станции?

Агент возвращает HTTP 503, а Viewer показывает машину как `Locked`, потому что secure desktop нельзя захватить из службы иначе.

Обязателен ли LLM-модуль?

Нет. Viewer работает как мониторинг-стена и без распознавания экрана; LLM — дополнительная локальная возможность.
