Core capability

A live monitoring wall instead of a loose collection of remote sessions.

Viewer polls every enabled machine and places the latest screenshots into an adaptive grid. Cards surface the real state immediately: normal screen, locked desktop, LLM status, load errors, or the absence of an active console session.

One operational view

Keep multiple Windows workstations in a single pane and open any screen in a zoom window on demand.

  • Adaptive tile wall for control rooms and medium-sized operator stations
  • Context actions directly from a computer card
  • Locked and disabled states stay visible without a separate diagnostics screen

Management inside Viewer

The same application handles the computer list, one-shot refreshes, model checks, and settings.

  • Bulk operations by selection or across the full enabled fleet
  • Refresh now and LLM now without changing the background cadence
  • Secure local storage or an encrypted shared connections file
Operations

Deployment and service maintenance are part of the operator workflow.

ScreensView is not limited to watching screens. Viewer helps you add machines in bulk, choose the right payload for the target Windows version, and service the agents without manually rebuilding on every workstation.

1

Add machines quickly

Single entry, host lists, or IP ranges. API keys can be generated automatically for new computers.

  • Host-list and IP-range entry modes
  • Expected screen description can be configured in the same workflow
2

Deploy agents remotely

Viewer copies the payload, creates the service, and chooses the modern or legacy agent for the target OS.

  • Windows 10/11 path plus Windows 7 SP1 compatibility branch
  • Step-by-step log for install, uninstall, and update operations

Keep the fleet current

Update agent services in bulk and trigger immediate screen verification when you need a faster answer.

  • Update all enabled computers without manual row selection
  • Viewer checks GitHub Releases for its own updates
Optional intelligence

LLM verification is an extra control layer, not the core product story.

If you need more than “there is a screenshot”, Viewer can compare the current screen with an expected layout and mark suspicious cards. The monitoring wall remains useful even when this feature is turned off.

Expected-screen verification

Describe the stable structure of the screen and let Viewer run local multimodal checks on a schedule or on demand.

  • Status badges `?`, `LLM`, `...`, `✓`, `✗`, `!` stay on the card itself
  • Frame deduplication avoids repeated LLM calls for identical screenshots
  • Manual checks work for the full wall or for one selected machine

Local model pipeline

Model files, backend binaries, and logs stay next to Viewer so diagnostics remain inside your environment.

  • Download compatible GGUF models from Settings
  • Separate diagnostics for `llama.cpp`, projector files, and unsupported architectures
  • Fast-profile image preprocessing for lower latency
Reliability and security

The showcase is lightweight, but the product is built around operational edge cases.

ScreensView accounts for Windows Service constraints, secure desktop behavior, and remote administration realities, not only the happy path where every screenshot is available at all times.

Explicit operational states

  • A locked workstation returns HTTP 503 and surfaces as `Locked` instead of pretending the machine is broken
  • Disabled computers drop out of polling and bulk actions immediately
  • `viewer.log` records cache hits, skipped frames, timeouts, and analysis failures

Minimal perimeter

  • TLS 1.2+, self-signed certificates, and thumbprint pinning in Viewer
  • Agent authorization through `X-Api-Key`
  • Remote deployment requires explicit local administrator access and `Admin$` / WMI reachability

FAQ

The short version before your first rollout.

This is not the full manual, only a compact overview for first-time visitors.

Do I need a cloud service or broker?

No. Viewer and the agent communicate directly over HTTPS inside your own network.

Is Windows 7 supported?

There is a legacy path for Windows 7 SP1 with `.NET Framework 4.8`, but it is a compatibility path, not a modern primary target.

What happens when a workstation is locked?

The agent returns HTTP 503 and Viewer marks the machine as `Locked`, because secure desktop capture is not available from the service in that state.

Is the LLM feature required?

No. The monitoring wall works without screen analysis; LLM verification is an optional local feature.
